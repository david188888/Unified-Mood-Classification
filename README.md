# Unified Mood Classification

Unified multitask learning for music mood classification. The code trains one model across two related tasks:

- DEAM valence/arousal regression
- MTG-Jamendo multi-label mood tag classification

The model combines MERT embeddings with mel-spectrogram, chroma, and tempogram features, then uses feature projection, fusion, a Conformer-style sequence encoder, and task-specific output heads.

## Main Files

- `train.py`: main training entry point
- `precompute_features.py`: offline feature extraction and cache generation
- `infer.py`: single-audio inference from a trained checkpoint
- `dataloader_fast.py`: cached-feature dataloaders used by training
- `generate_mtg_csv.py`: builds the curated MTG-Jamendo `mtg_labels.csv` from local MTG split metadata
- `compute_r.py`: recomputes DEAM `R^2` values from checkpoints or prediction tensors
- `src/models/`: model, fusion, projection, encoder, and output-head modules
- `src/losses/`: multitask loss implementation

## Setup

Use Python 3.11 or newer. The project has been run with a Conda environment named `music`.

```bash
conda create -n music python=3.11 -y
conda activate music
pip install -r requirements.txt
```

## External Resources

Large resources are intentionally not stored in this repository:

- DEAM audio and annotation files
- MTG-Jamendo audio and split metadata
- the pretrained `MERT/` model directory
- generated feature caches
- checkpoints and training outputs under `runs/`

Place those resources locally before running data preparation, feature extraction, training, or inference.

## Expected Layout

```text
Unified-Mood-Classification/
├── data/
│   ├── DEAM/
│   │   ├── audio/
│   │   ├── annotations/annotations averaged per song/song_level/
│   │   └── deam_split.json
│   ├── MTG-Jamendo/
│   │   ├── <numeric audio subfolders>/
│   │   ├── mtg_split_labels.csv
│   │   └── mtg_labels.csv
│   └── features/
│       ├── deam/
│       └── mtg/
├── MERT/
└── runs/
```

`data/features/` is generated by `precompute_features.py`. `runs/` is generated by `train.py`. `MERT/` should contain the local pretrained MERT files loaded through `transformers`.

## Data Preparation

The `data/` directory is ignored by git. Prepare these local files before feature extraction:

- `data/DEAM/deam_split.json`
- `data/MTG-Jamendo/mtg_split_labels.csv`
- DEAM and MTG-Jamendo audio files in the layout shown above

After `data/MTG-Jamendo/mtg_split_labels.csv` and the MTG audio files are in place, build the curated MTG label CSV used by the dataloader:

```bash
python generate_mtg_csv.py
```

The training pipeline expects `data/MTG-Jamendo/mtg_labels.csv` with `split`, `track_id`, `mood_tags`, and `audio_path` columns.

## Typical Workflow

### 1. Precompute Features

Training uses cached features by default.

```bash
python precompute_features.py --dataset all --model_dir MERT
```

Dataset options are `deam`, `mtg-jamendo`, and `all`. Cached feature files contain:

- `mert`
- `mel`
- `chroma`
- `tempogram`

### 2. Train

```bash
# Early fusion
python train.py --fusion_type early --batch_size 8 --epochs 50

# Late fusion
python train.py --fusion_type late --batch_size 8 --epochs 50
```

Useful options:

- `--enabled_features mert,mel,chroma,tempogram`
- `--train_pct 0.1 --subset_seed 42`
- `--resume <checkpoint_path>`
- `--no_auto_resume`
- `--num_workers <N>`
- `--eval_num_workers <N>`
- `--plain_progress`
- `--compile`

Current feature constraints:

- early fusion requires `mert`
- late fusion currently requires the full feature set

### 3. Run Inference

```bash
python infer.py \
  --audio_path path/to/audio.mp3 \
  --checkpoint path/to/checkpoint_best.pt \
  --task both \
  --model_dir MERT
```

`--task` supports `auto`, `deam`, `mtg`, and `both`. Inference first uses cached features for known dataset samples; if no cache is available, it extracts features with the local MERT model.

### 4. Plot Metrics

```bash
python scripts/plot_metrics.py --metrics runs/unified_mood_model_early/metrics.csv
```

The repository tracks code and lightweight helper scripts only. Local datasets, MERT weights, generated features, checkpoints, training outputs, and `.DS_Store` files are ignored.
